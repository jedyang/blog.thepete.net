--- layout: post title: "Essential reads for a software developer" date: 2011-01-09T15:37:00-08:00 comments: false ---

Pragmatic Programmer: Covers basic low-level design best practices as well as many of the foundational practices all developers should be following (testing,version control,refactoring).

Code Complete: Similar content to The Pragmatic Programmer. More indepth, a little more reasoned but also a little drier.

Growing Object-Oriented Software Guided By Tests The main focus of this book is about building software using the ‘London School’ of TDD, but along the way it describes the kind of advanced OO design which takes years to fully grok.

Refactoring: Thorough and readable description of a core developer practice.

Clean Code: Opinionated but full of very useful guidance on OO design.

Test-Driven Development: By Example: Maybe a little dated now, but a great introduction to the foundational principle of TDD

Extreme Programming Explained: A little dated at this point (although I haven’t read the 2nd edition). Provides a nice short intro to agile practices couched in relatively developer-centric terms.

Refactoring to Patterns A higher level look at ways to apply the practice of refactoring to evolve towards Patterns. A good way to solidify your understanding of the practice of refactoring.

RESTful Web Services: Pragmatic but principled coverage on building RESTful services.

Pragmatic Thinking & Learning: Refactor Your Wetware: The only non-technical book in my list, but one of the more valuable. Your entire career revolves around problem solving and interacting with people. Get better at both.

xUnit Test Patterns: Encyclopedic coverage of unit testing patterns and practices.

Working Effectively with Legacy Code: Absolutely required reading for every developer on how to maintain and extend existing systems.

Patterns of Enterprise Application Architecture: Full to the brim with higher-level design/architecture patterns, some of which are applicable to most any large software systems.

Pattern-Oriented Software Architecture (POSA) Volume 1, Volume 2, Volume 4: Rather dated, and a bit dry and academic, but some often-overlooked gems can be found in here. For example, lots of coverage of event-driven patterns now made trendy by Node.js, Event Machine et al.

Object-Oriented Software Construction: Old and fairly academic, but lots of stuff that’s still relevant. Meyer’s Design By Contract approach is definitely something more developers should be exposed to.

Implementation Patterns: A straightforward explanation of the low-level code design tricks which seasoned developers learn through years of practices. You will apply lessons from this book on a daily basis.

Head First Design Patterns: While the original Gang of Four Design Patterns book is obviously the canonical source, this book presents the core patterns in a more digestable, modern form.

Domain Driven Design: This is really two books in one. The first half covers useful mid-to-high level ‘tactical’ issues, introducing a set of highly-regarded design patterns. The second half covers high-level ‘strategic’ issues which span teams and multiple co-dependent code bases.

Release It!: Written long before the DevOps movement took off, this is a great resource for developers who care about the production life of the systems they help create.

Structure and Interpretation of Computer Programs (SICP): I haven’t finished reading it, but I feel like I should have. Subtle, math-heavy, and mind-altering.
